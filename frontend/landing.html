on git.push where event.branch is "main"

Compare that to GitHub:
GitHub Actions 
name: Test

on:
  push:
    branches:
      - "main"

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Run tests
        run: |
          npm install
          npm run test
Gitlab CI 
stages:
  - test

run-tests:
  stage: test
  script:
    - npm install
    - npm run test
  rules:
    - if: $CI_COMMIT_BRANCH == "main"

Use Your Favorite Git Providers

Cicada supports both GitHub and Gitlab. Azure DevOps, Bitbucket and many more coming soon.

No Vendor Lock-in

  • Triggers in Cicada are platform agnostic, meaning a git.push trigger coming from GitHub will look the same as one from Gitlab.
  • Similar to GitHub Actions, Cicada supports many different triggers types including issue.open, issue.close, git.push, and more.
  • Say goodbye to porting your workflows from platform to platform!
